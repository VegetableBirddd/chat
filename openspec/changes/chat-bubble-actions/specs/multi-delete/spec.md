## ADDED Requirements

### Requirement: 进入多选删除模式
用户点击删除按钮后，当前会话进入多选删除模式。

#### Scenario: 点击删除进入多选
- **WHEN** 用户点击删除按钮
- **THEN** 会话进入多选删除模式
- **THEN** 所有消息（用户和 AI）左侧出现勾选框
- **THEN** 输入框区域变为确认删除按钮，显示"删除 X 条消息"

#### Scenario: 多选模式显示已选数量
- **WHEN** 进入多选模式且已选中消息
- **THEN** 底部固定区域显示"已选择 X 条消息"

### Requirement: 选择消息
在多选模式下，用户可以勾选当前会话内的任意消息。

#### Scenario: 勾选用户消息
- **WHEN** 用户点击用户消息的勾选框
- **THEN** 该消息被标记为选中状态
- **THEN** 底部已选数量更新

#### Scenario: 勾选 AI 消息
- **WHEN** 用户点击 AI 消息的勾选框
- **THEN** 该消息被标记为选中状态
- **THEN** 底部已选数量更新

#### Scenario: 取消勾选
- **WHEN** 用户再次点击已选中的勾选框
- **THEN** 该消息取消选中状态
- **THEN** 底部已选数量更新

### Requirement: 确认删除
用户确认删除选中的消息。

#### Scenario: 确认删除
- **WHEN** 用户点击底部确认删除按钮
- **THEN** 所有选中的消息从 ChatStore 中移除
- **THEN** 退出多选删除模式
- **THEN** 输入框恢复为正常输入状态

#### Scenario: 至少选择一条消息
- **WHEN** 已选消息数量为 0 时点击确认删除
- **THEN** 按钮为禁用状态，提示"请选择要删除的消息"

### Requirement: 取消删除模式
用户可以取消多选删除模式而不执行任何删除操作。

#### Scenario: 取消多选模式
- **WHEN** 用户点击取消按钮或按 ESC 键
- **THEN** 退出多选删除模式
- **THEN** 勾选框隐藏
- **THEN** 输入框恢复为正常输入状态

### Requirement: 多选状态管理
多选删除模式的状态由 ChatStore 统一管理。

#### Scenario: Store 多选状态
- **WHEN** 会话进入多选删除模式
- **THEN** Store 记录多选模式状态及选中的消息 id 列表
- **THEN** InputBox 组件读取状态显示确认栏
