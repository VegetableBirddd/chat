## Context

- **现状**：当前 ChatMessage.vue 只显示消息内容和时间，无操作功能
- **用户需求**：Hover 用户消息气泡时显示删除、编辑、复制按钮
- **约束**：纯图标按钮 + tooltip，纯用户消息可操作，仅当前会话内多选删除

## Goals / Non-Goals

**Goals:**
- 用户 hover 自己的消息气泡时显示操作按钮
- 点击编辑进入 inline 编辑模式，编辑完成后可保存或取消
- 点击复制复制纯文本内容到剪贴板
- 点击删除进入多选模式，可选择会话内任意消息删除
- 删除需确认，确认栏固定在输入框上方

**Non-Goals:**
- 不支持 AI 消息的操作（仅用户消息可操作）
- 不支持会话间批量删除
- 不支持消息的移动/排序功能

## Decisions

### D1: 状态管理方案
**选择**: Store 集中管理（ChatStore）

**理由**: 
- 多选模式需要在 InputBox 显示确认栏，需要跨组件共享状态
- 编辑状态可能需要取消操作，需要全局控制
- 符合用户确认的方案 B

### D2: 操作按钮样式
**选择**: 纯图标 + hover tooltip 显示中文提示

**理由**:
- 符合用户需求 "纯图标按钮，hover上去有个中文提示框"
- 保持界面简洁

### D3: 删除多选模式 UI
**选择**: 底部固定在输入框上方

**理由**:
- 用户明确选择 "底部固定（推荐）"
- 输入框上方符合常见交互习惯

### D4: 删除确认流程
**选择**: 进入多选状态 → 勾选消息 → 点击删除按钮 → 确认删除

**理由**:
- 用户需求：点击删除时进入多选状态，可以对用户和AI的对话进行选择
- 输入框区域变为确认删除按钮

## Risks / Trade-offs

- [风险]: inline 编辑时用户离开页面未保存
  - [缓解]: 编辑状态在 Store 中管理，页面离开时提示保存
- [风险]: 多选删除误操作重要消息
  - [缓解]: 需要二次确认，勾选后底部显示已选数量
