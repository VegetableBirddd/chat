## ADDED Requirements

### Requirement: 用户消息发送失败后可以重新发送
当用户消息发送失败时，系统 SHALL 在该消息气泡的左侧显示重新发送按钮，点击后可以重新发送该消息。

#### Scenario: 显示重新发送按钮
- **WHEN** 用户发送消息后 API 请求失败
- **THEN** 消息气泡左侧显示重新发送图标按钮

#### Scenario: 重新发送成功
- **WHEN** 用户点击重新发送按钮
- **THEN** 消息重新发送，发送成功后移除失败状态
- **AND** 界面显示正常的助手回复

#### Scenario: 重新发送失败
- **WHEN** 用户点击重新发送按钮后再次失败
- **THEN** 消息保持失败状态
- **AND** 可以再次尝试重新发送

### Requirement: 重新发送时显示加载状态
- **WHEN** 用户点击重新发送按钮后消息正在发送中
- **THEN** 重新发送按钮显示加载动画或变为禁用状态

### Requirement: 仅用户消息显示重新发送按钮
- **WHEN** 助手消息发送失败
- **THEN** 不显示重新发送按钮
- **AND** 助手消息的失败由系统自动处理或用户通过重新输入触发
