## Context

当前聊天应用中，用户发送消息后如果 API 请求失败（例如网络超时、API 错误），消息会显示为失败状态，但用户无法直接从界面上重新发送该消息。需要在 UI 上提供重新发送的操作入口。

现有架构：
- `ChatMessage.vue` 渲染用户和助手的消息气泡
- `chat.ts` (Pinia store) 管理消息状态和 API 调用
- `api.ts` 处理 DeepSeek API 请求

## Goals / Non-Goals

**Goals:**
- 在发送失败的用户消息气泡左侧显示重新发送图标
- 点击图标后触发消息重新发送
- 重新发送过程中显示加载状态
- 发送成功后移除失败状态标记

**Non-Goals:**
- 不实现消息编辑功能
- 不实现批量重发功能
- 不修改系统消息的失败处理逻辑

## Decisions

1. **在消息对象中添加 `error` 字段标识失败状态**
   - 理由：现有的消息对象只有 `content`, `role`, `id` 等基础字段，需要扩展以支持失败状态和重试
   - 替代方案：在 store 中维护一个 `failedMessageIds` 集合 -考虑到状态一致性，选择直接在消息对象上标记

2. **重新发送按钮使用 SVG 图标而非图标库**
   - 理由：避免引入额外依赖，保持项目轻量
   - 替代方案：使用 iconify-icon - 需要安装额外包

3. **失败状态使用 `error` 字段（字符串，存储错误信息）而非布尔值**
   - 理由：错误信息可用于展示详细的失败原因
   - 替代方案：仅使用布尔值 - 不够灵活

## Risks / Trade-offs

- [风险] 如果消息发送失败但网络恢复后自动重试 - 考虑：不实现自动重试，仅提供手动触发
- [风险] 重新发送时消息 ID 是否保持一致 - 决策：使用新 ID 发送，避免 API 侧缓存问题
